# Paimon Live — 产品需求文档

---

## 1. 这个产品是什么

Paimon Live 是一个**由大模型驱动的虚拟 AI 主播桌面系统**。

它将 Live2D 虚拟角色与 AI 对话能力整合，形成一个可用于直播场景的桌面应用。操作者可以通过控制台监控和干预直播流程，让非技术用户也能完成基本操作。

它**不是**：
- VoiceL2D 的简单桌面壳
- 一个技术 demo 的打包版本
- 一个需要手动启动多个服务的开发环境
- 一个云平台或 SaaS 产品

---

## 2. 谁在使用

### 主要用户：直播操作员

- 可能是主播本人，也可能是助理/运营人员
- 技术水平不确定——系统应降低使用门槛
- 需要在直播过程中实时监控、干预角色行为
- 需要一键启动，不需要理解底层服务架构

### 次要用户：配置管理员

- 负责角色人设编辑、商品信息更新、知识库维护
- 需要配置界面或配置文件编辑能力
- 可以接受更高的技术门槛

### 开发者

- 负责系统部署、外部服务对接、功能扩展
- 需要完善的日志、调试工具和文档

---

## 3. 核心使用场景

### 场景 1：AI 虚拟主播直播

操作员启动 Paimon Live，角色自动开始在直播中互动：
- 响应观众语音/弹幕
- 按角色人设回复
- 自动触发表情和动作
- OBS 捕获透明背景的角色画面

### 场景 2：操作员监控与干预

直播过程中操作员通过控制面板：
- 实时查看系统状态和对话内容
- 手动触发特定表情/动作
- 在 AI 回复不当时急停或人工接管
- 切换主推商品、注入临时话术

### 场景 3：直播前准备

操作员/管理员在直播前：
- 配置角色人设和 TTS 声线
- 录入当场直播的商品信息和话术
- 测试 AI 回复效果
- 检查各服务连接状态

## 4. 核心能力需求

| 编号 | 能力 | 优先级 | 说明 |
|------|------|--------|------|
| C1 | Live2D 角色渲染 | 必须 | 角色在窗口中显示，支持表情和动作 |
| C2 | 语音链路 | 必须 | ASR → LLM → TTS 完整闭环 |
| C3 | 表情/动作触发 | 必须 | LLM 工具调用驱动角色行为 |
| C4 | 控制面板 | 必须 | 操作员监控和干预界面 |
| C5 | 急停与人工接管 | 必须 | 保证直播安全 |
| C6 | OBS 透明输出 | 必须 | 供直播软件捕获 |
| C7 | 外部事件接入 | 重要 | 弹幕、礼物、商品消息 |
| C8 | 知识/上下文管理 | 重要 | 长期知识 + 临时高优先级消息 |
| C9 | 日志与调试 | 重要 | 问题排查和系统监控 |
| C10 | 一键启动 | 重要 | 非技术人员可操作 |
| C11 | 打包分发 | 重要 | Windows 安装包 |

---

## 5. 当前明确不做什么

| 事项 | 原因 |
|------|------|
| macOS / Linux 支持 | 当前仅 Windows，未来再扩展 |
| 完整 RAG 系统 | 只预留接口，不实现向量检索 |
| SaaS / 多人协作 | 单机桌面应用 |
| 虚拟摄像头 | 使用 OBS 窗口捕获替代 |
| 本地 ASR（faster-whisper） | 云端 ASR 为默认路径，本地 ASR 作为可选延后 |
| 多角色同时在线 | 当前设计为单角色 |
| 自动弹幕筛选/排队 | 可作为后续增强 |
| 性能优化 | 先完成功能，再做性能 |

---

## 6. 与 VoiceL2D 的区别

| 维度 | VoiceL2D | Paimon Live |
|------|----------|-------------|
| 定位 | 技术原型 / demo | 可操作的桌面产品 |
| 架构 | 三个独立进程手动启动 | 单桌面宿主统一管理 |
| 操作者 | 开发者本人 | 非技术直播操作员 |
| 监控 | 无 | 控制面板实时监控 |
| 安全 | 无急停机制 | 急停 + 人工接管 |
| 直播集成 | 需手动配合 OBS | 透明窗口直出 |
| 分发 | 源码运行 | Windows 安装包 |
| 知识管理 | 无 | 长期知识 + 临时上下文 |

---

## 7. 成功标准

### 最小可行产品（MVP）标准

- [ ] 操作员启动一个桌面程序即可开始直播
- [ ] 角色能响应语音输入并做出语音+表情回复
- [ ] OBS 能捕获透明背景的角色画面
- [ ] 操作员能在控制面板中监控系统状态
- [ ] 操作员能急停和人工接管
- [ ] 能注入商品信息影响 AI 回复内容

### 体验标准

- [ ] 非技术人员能在 10 分钟内完成首次配置并启动
- [ ] 系统运行 2 小时以上不崩溃
- [ ] 语音链路延迟在可接受范围内（端到端 < 5 秒）
- [ ] 控制面板信息清晰，操作员不需要查看终端日志
