## 2. 关键改动

### 2.1 Stage 置顶遮挡的真正根因

**根因**：`src-tauri/tauri.conf.json` 中 stage 窗口配置了 `"parent": "main"`。

```
┌─────────────────────────────────┐
│ Main Window                     │
│  ┌───────────────────────────┐  │
│  │ Settings Panel (z-index:1300) │  ← 被遮挡
│  └───────────────────────────┘  │
└─────────────────────────────────┘
        ▲ parent 关系
┌─────────────────────────────────┐
│ Stage 窗口 (child)               │
│  always-on-top: true            │  ← 永远显示在 main 上方！
└─────────────────────────────────┘
```

子窗口（child window）在 OS 窗口层级中**永远**位于父窗口之上，与 `setAlwaysOnTop` 值无关。因此之前所有 `suppress` / `restore` 逻辑在有 parent 约束的子窗口上根本无法生效。

**修复**：移除 stage 窗口的 `"parent": "main"` 约束，使 `setAlwaysOnTop(false)` 能真正将 stage 降到 main 窗口下方。

> 注意：main window close 时 stage 的关闭逻辑在 Rust 端 `lib.rs` 的 `on_window_event` 中通过遍历 `webview_windows()` 实现，不依赖 parent 关系，移除后功能不受影响。

### 2.2 Stage 置顶恢复竞态修复

配合 2.1 的修复，引入 `suppress-always-on-top` 命令，`StageWindow` 用 `suppressedRef` 记住 suppression 前的值，cleanup 时用 ref 值恢复。

**涉及文件**：
- `src-tauri/tauri.conf.json` — 移除 `parent: "main"`
- `src/utils/window-sync.ts` — 新增 `suppress-always-on-top` 命令
- `src/features/stage/StageWindow.tsx` — 新增 `suppressedRef`，实现 `suppress` / `restore` 分支
- `src/features/settings/SettingsPanel.tsx` — 发送 `suppress` → `restore` 序列
