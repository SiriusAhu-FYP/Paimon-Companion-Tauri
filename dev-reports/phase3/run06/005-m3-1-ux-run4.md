# M3.1 UX Run 4 — 问题修复与目录整理

## 1. 本次完成了什么

修复了 4 个反馈问题：

1. **Stage 置顶恢复竞态** — 之前 `SettingsPanel` 的 `useEffect` cleanup 发 `restore-always-on-top` 时，依赖数组为空但 closure 捕获的 `alwaysOnTop` state 已经被 `sync-state` 更新为 `false`，导致恢复值错误。

## 2. 关键改动

### 2.1 Stage 置顶恢复竞态修复

**根因**：`SettingsPanel` 卸载时，`StageWindow` 的 `alwaysOnTop` state 已经被中间的 `sync-state` 广播更新为 `false`（因为 Stage 实际置顶状态已被降级），导致 `restore-always-on-top` 分支中的 `alwaysOnTop` 参数等于 `false`。

**修复方案**：引入 `suppress-always-on-top` 命令，`StageWindow` 收到后：
- 用 `suppressedRef` 记住 suppression 前的值
- 将窗口置顶设为 `false`
- 收到 `restore-always-on-top` 时，用 `suppressedRef` 中的值恢复，不再依赖 state closure

**涉及文件**：
- `src/utils/window-sync.ts` — 新增 `suppress-always-on-top` 命令
- `src/features/stage/StageWindow.tsx` — 新增 `suppressedRef`，实现 `suppress` / `restore` 分支
- `src/features/settings/SettingsPanel.tsx` — 发送 `suppress` → `restore` 序列
